# A股股票数据库 - 整体设计方案

> 版本: v1.0
> 日期: 2026-04-07
> 状态: 待评审

---

## 一、设计目标

1. **全量存储** - 从akshare获取的数据全部存入数据库，保留原始字段
2. **可追溯** - 每个字段标记对应的akshare原始名称和含义
3. **实时分时** - 支持1分钟级别分时数据，滚动保留近5个交易日
4. **数据隔离** - 历史数据与实时数据分离，便于维护

---

## 二、技术架构

| 组件 | 选型 | 说明 |
|------|------|------|
| 数据库 | SQLite | 轻量级、零配置、本地存储 |
| ORM | SQLAlchemy 2.0 | 成熟的Python ORM，支持事务 |
| 数据源 | akshare | 免费开源，东方财富/新浪/腾讯等 |
| 分时获取 | 定时任务 + 日结归档 | 1分钟数据定时写入 |
| 数据清理 | 自动清理 | 保留近5个交易日 |

### 3.3 stock_minute - 1分钟分时数据

**数据来源**:
- 历史数据（非当日）: `akshare.stock_zh_a_minute(symbol=code, period="1", adjust="qfq")`
- 当日数据: `akshare.stock_intraday_sina(symbol=code, date="YYYYMMDD")` + 按分钟聚合

| 字段名 | 类型 | 原始字段 | 含义 | 说明 |
|--------|------|----------|------|------|
| id | INTEGER | - | 自增ID | 主键 |
| code | VARCHAR(10) | - | 股票代码 | 索引 |
| 日期 | DATETIME | - | 交易日期 | 分区字段 |
| 时间 | DATETIME | 时间 | 精确时间 | akshare原始字段 |
| 开盘 | DECIMAL(10,3) | 开盘 | 开盘价 | akshare原始字段 |
| 收盘 | DECIMAL(10,3) | 收盘 | 收盘价 | akshare原始字段 |
| 最高 | DECIMAL(10,3) | 最高 | 最高价 | akshare原始字段 |
| 最低 | DECIMAL(10,3) | 最低 | 最低价 | akshare原始字段 |
| 成交量 | DECIMAL(20,2) | 成交量 | 成交量 | akshare原始字段 |
| 成交额 | DECIMAL(20,2) | 成交额 | 成交额 | 当日数据不含 |
| raw_data | JSON | - | 原始JSON | 全量备份，包含数据来源 |

**数据获取逻辑**:
1. 获取历史分时数据（stock_zh_a_minute），过滤掉当日数据
2. 获取当日逐笔数据（stock_intraday_sina），按分钟聚合OHLC
3. 合并写入数据库

**数据保留策略**:
- 仅保留近5个交易日的1分钟数据
- 每日收盘后自动清理超过5日的数据

**索引**:
- UNIQUE(code, 时间)
- INDEX(code, 日期)

**数据量估算**:
- 单股每日: ~240条（扣除休息时间）
- 单股5日: ~1200条
- 5000股5日: ~600万条 (~200MB)

## 四、数据获取策略

### 4.1 定时任务配置

| 任务 | 频率 | 数据范围 | 说明 |
|------|------|----------|------|
| 股票基本信息 | 每日1次 | 全量 | 盘后更新 |
| 日线行情 | 每日收盘后 | 全量/增量 | 复权数据 |
| 1分钟分时 | 交易时间内 | 全量/增量 | 5日滚动 |
| 实时行情 | 每5分钟 | 全量 | 当日快照 |
| 财务数据 | 每季度 | 增量 | 财报发布后 |
| 指数成分股 | 每周1次 | 全量 | 定期更新 |
| 概念板块 | 每30分钟 | 全量 | 热点追踪 |

### 4.2 交易时间

| 时段 | 时间 | 分时数据 |
|------|------|----------|
| 早盘集合竞价 | 09:15-09:25 | 无 |
| 连续竞价 | 09:30-11:30 | 1分钟K |
| 午间休市 | 11:30-13:00 | 无 |
| 连续竞价 | 13:00-15:00 | 1分钟K |
| 尾盘集合竞价 | 15:00-15:05 | 无 |

**每日有效分时**: 09:30-11:30, 13:00-15:00（约4小时240条）

### 4.3 分时数据获取流程

```
交易日 09:30 开始
    ↓
每分钟定时任务
    ↓
获取所有股票1分钟数据
    ↓
清洗后写入 stock_minute
    ↓
每日收盘后清理 >5日 数据
```

## 五、数据清理策略

### 5.1 分时数据清理

```python
# 保留近5个交易日
CLEANUP_BEFORE = 当前日期 - 5交易日

DELETE FROM stock_minute
WHERE 日期 < CLEANUP_BEFORE
```

### 5.2 清理时机

- 每日收盘后（15:30）执行一次
- 或者每日凌晨执行

## 九、数据溯源机制

### 9.1 字段对照

每个数据库字段都标记了对应的akshare原始字段名，例如：

```python
# 数据库字段: 开盘 (DECIMAL)
# 对应 akshare 字段: 开盘
# 含义: 开盘价(元)
# 数据来源: stock_zh_a_hist()
```

### 9.2 原始数据备份

```python
raw_data = Column(JSON)  # 每个表都有

# 存储示例
{
    "source": "akshare.stock_zh_a_hist",
    "fetch_time": "2026-04-07 10:30:00",
    "original_data": {...}  # 原始完整JSON
}
```

### 9.3 问题排查流程

```
发现数据异常
    ↓
根据字段名 → 找到原始字段来源
    ↓
查询 raw_data → 还原原始数据
    ↓
对比分析 → 定位问题
```
